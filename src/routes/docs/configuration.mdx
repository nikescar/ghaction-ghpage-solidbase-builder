# Configuration

Learn how to configure Jekyll Solidbase Theme to match your needs.

## Basic Configuration

The main configuration is done in your `_config.yml` file:

```yaml
# Site settings
title: Your Site Title
description: Your site description

# Theme settings
theme_settings:
  social_links:
    - GitHub: 'https://github.com/your-username'
  en:
    title: Your Site Title
    nav:
      - Docs: /docs
      - Blog: /blog
    sidebar:
      - Docs:
          - Getting Started: /docs/getting-started
          - Features: /docs/features
          - Configuration: /docs/configuration
```

## Navigation Configuration

### Simple Navigation
For simple navigation items without custom links:

```yaml
nav:
  - Docs    # Automatically becomes /docs
  - Blog    # Automatically becomes /blog
```

### Custom Navigation
For navigation with custom links:

```yaml
nav:
  - Docs: /documentation
  - Blog: /articles
  - About: /about-us
```

## Sidebar Configuration

### Auto-Discovery
For automatic sidebar generation from markdown files:

```yaml
sidebar:
  - Docs    # Auto-discovers all .md files in ./docs/
```

### Manual Configuration
For manual sidebar structure:

```yaml
sidebar:
  - Docs:
      - Getting Started: /docs/getting-started
      - Advanced: /docs/advanced
      - FAQ: /docs/faq
```

## Social Links

Configure social media links:

```yaml
theme_settings:
  social_links:
    - GitHub: 'https://github.com/username'
    - Twitter: 'https://twitter.com/username'
    - LinkedIn: 'https://linkedin.com/in/username'
```

## Build Configuration

The theme uses Vinxi for building. You can customize the build process in `app.config.ts`:

```typescript
export default defineConfig(
  createWithSolidBase(defaultTheme)(
    {
      ssr: false,
      server: {
        preset: "static",
        prerender: {
          routes: ["/", "/docs", "/blog"],
          crawlLinks: true,
        },
      },
    },
    // Theme configuration from _config.yml is loaded here
  )
);
```
